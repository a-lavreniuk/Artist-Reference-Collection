# Закрыть задачу в AnyType

Финальный шаг после **git-task-finish** (или вместо него, если git не нужен): обновить или создать карточку на доске **Задачи** / **Баг-репорты**.

AnyType Desktop должен быть запущен. MCP: `user-anytype`. Константы — [anytype-ids.md](references/anytype-ids.md).

## Когда выполнять

| Ситуация | Действие |
|----------|----------|
| Git-финал прошёл успешно | Обновить карточку AnyType |
| Git не требовался (только skills/docs/анализ) | Создать или обновить карточку AnyType |
| Git остановлен (тесты, конфликт, critical review) | AnyType **не трогать**, сообщить пользователю |
| Пользователь настаивает закрыть при незавершённой работе | «Готово» + в «Результат» явно указать, что не доделано |

## 1. Определить карточку

**Приоритет:**

1. **`object_id` из сессии** — запомнен при **anytype-task-take** («возьми задачу»). Это основной способ привязки.
2. **Нет `object_id`** — задача дана в чате → **создать новую карточку** (п. 4).
3. **`object_id` потерян**, но задача точно из AnyType → спросить пользователя, какую карточку закрыть. **Не угадывать** по поиску.

## 2. Сформировать секцию «Результат»

Добавить в конец markdown-тела карточки (не заменять исходное описание):

```markdown
## Результат

[Ссылка на PR, если есть — первая строка]

5–10 предложений о проделанной работе: что изменилось для пользователя, ключевые файлы/области, как проверить. Если работа неполная — отдельным абзацем перечислить, что осталось недоделанным.
```

Правила:

- Заголовок секции — **`## Результат`** (ровно так).
- **Ссылка на PR** — если PR создан; иначе ссылка на merge-коммит в GitHub или явная пометка «PR не создавался».
- Тон: деловой, по делу, без эмодзи.
- Если секция «Результат» уже есть — **дописать** новый блок с датой или заменить содержимое под заголовком (не дублировать заголовок).

## 3. Обновить существующую карточку

1. `API-get-object` (`format: md`) — прочитать текущий markdown.
2. `API-update-object`:
   - `markdown` — исходный текст + секция «Результат»;
   - `properties` — **только** `{ "key": "status", "select": "<tag id Готово>" }`.

**Не менять:** Cursor, Source, тип обращения, «Насколько мешает», имя карточки и прочие свойства.

## 4. Создать карточку (задача из чата)

Если карточки в AnyType не было:

1. **Спросить тип:** «Задача» (`task`) или «Баг-репорт» (`bagi`).
2. **Название** — сформировать самому, 2–3 слова по сути работы.
3. **Тело:**
   - **Задача (`task`):** исходный запрос из чата + секция «Результат».
   - **Баг-репорт (`bagi`):** шаблон как в существующих записях:

```markdown
## Описание
 ---
<исходное описание проблемы/запроса из чата>

## Результат

<ссылка на PR + 5–10 предложений>
```

4. `API-create-object`:
   - `type_key`: `task` или `bagi`;
   - `name`, `body` (markdown);
   - `properties`: только `status` → «Готово»;
   - `source` (url) — если в чате была релевантная ссылка (Figma, статья, issue);
   - поле **Cursor не заполнять**.

## 5. Отчёт пользователю

После успешного обновления/создания:

- название карточки;
- тип (Задача / Баг-репорт);
- статус «Готово»;
- кратко — что записано в «Результат».

## 6. Связка с другими skills

- Старт и запоминание `object_id` — **anytype-task-take**.
- Git (commit, push, merge) — **git-task-finish** (шаги 1–5b), **до** этого skill.
- Порядок при «закрой задачу»: git-task-finish → anytype-task-finish.

## Примеры

**Из AnyType:** «возьми задачу» → работа → «закрой задачу» → git merge → `API-update-object` по сохранённому `object_id`, статус «Готово», в конец тела — «Результат» с PR.

**Из чата:** «сделай X» → работа без ветки → «закрой задачу» → спросить task/bagi → `API-create-object` сразу в «Готово».

**Неполная работа:** пользователь просит закрыть → «Готово» + в «Результат»: «Не завершено: …».
