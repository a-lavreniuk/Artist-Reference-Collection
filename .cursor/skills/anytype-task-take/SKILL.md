# Взять задачу из AnyType

Полуавтоматический старт работы: пользователь вручную переносит карточку в колонку **«В работе»** и говорит «возьми задачу» — агент читает задачу в AnyType и действует по режимам поля **Cursor**.

AnyType Desktop должен быть запущен. MCP: `user-anytype`. Константы — [anytype-ids.md](../anytype-task-finish/references/anytype-ids.md).

## 1. Прочитать задачу

1. `API-list-spaces` → space **Artist Reference Collection** (или единственный space).
2. `API-search-space` с `types: ["task", "bagi"]` → карточки со **Статус** = `В работе` (доски **Задачи** и **Баг-репорты**).
3. **Одна** карточка в «В работе» → брать её. **Несколько** → спросить, какую. **Ноль** → сообщить и остановиться.
4. `API-get-object` (`format: md`) → название, markdown-тело, свойства, `type.key` (`task` / `bagi`).
5. Поле **Cursor** (`key: cursor`, `multi_select`) — теги: `Нужна ветка`, `Нужен план`, `Хотфикс`.
6. **Cursor пустой** → спросить режим, **не начинать** без ответа.

**Запомнить на всю сессию** (для **anytype-task-finish** при «закрой задачу»):

- `anytype_object_id` — id карточки;
- `anytype_type_key` — `task` или `bagi`;
- `anytype_object_name` — название.

Кратко пересказать задачу пользователю: название, тип (задача / баг-репорт), статус, активные режимы Cursor, суть из описания.

## 2. Приоритеты режимов

| Правило | Действие |
|---------|----------|
| **«Нужна ветка»** | Высший приоритет. **Никогда не писать код в `main`**, если этот тег есть |
| **«Нужна ветка» + «Нужен план»** | Сначала ветка, потом Plan mode |
| **Только «Нужен план»** | `main` без новой ветки → Plan mode |
| **«Хотфикс»** (без «Нужна ветка») | Agent mode, сразу к реализации |

Если есть и «Нужна ветка», и «Хотфикс» — **ветка побеждает**, hotfix не отменяет ветку.

## 3. Режим «Нужна ветка»

```bash
git checkout main
git pull origin main
git checkout -b <branch-name>
```

**Имя ветки:** перевести название задачи на английский, kebab-case, **без префикса** (`feature/` не нужен).

Пример: «Необратимые удаления» → `irreversible-deletions`.

После создания ветки вызвать `SetActiveBranch` с новым именем.

Если **нет** «Нужен план» → **остановиться**, ждать «делай» / «реализуй».

## 4. Режим «Нужен план»

1. Если ещё не на нужной ветке и нужна ветка — сначала п.3.
2. `SwitchMode` → **plan**.
3. Проанализировать задачу (описание, контекст ARC, связанные файлы).
4. Задать вопросы, пока не останется пробелов в понимании.
5. **Не писать код** до явной команды «делай» / «реализуй» / одобрения плана.

Если только «Нужен план» (без ветки) → `git checkout main` (pull не обязателен, если ветка не создаётся).

## 5. Режим «Хотфикс»

Только если **нет** «Нужна ветка»:

- Остаться в **Agent mode**.
- Приступить к реализации; вопросы — по необходимости.
- Обычно единичная правка на текущей ветке или `main`.

## 6. После «делай» / «реализуй»

- `SwitchMode` → **agent** (если был Plan).
- Реализовать по задаче и правилам ARC.
- UI-задачи — дополнительно skill **arc-ui-dev**.
- Закрытие: **git-task-finish** → **anytype-task-finish** по команде «закрой задачу» / «заверши задачу».

## 7. Ограничения

- Не коммитить и не пушить без явной просьбы.
- **Не менять статус** карточки в AnyType при старте — в «В работе» переносит только пользователь.
- Не создавать ветку, если пользователь назвал задачу вручную в чате, но не сказал «возьми задачу» — уточнить.

## Примеры

**Ветка + план:** «Возьми задачу» → Cursor: `Нужна ветка`, `Нужен план` → `irreversible-deletions` → Plan → вопросы → стоп → пользователь: «делай» → код.

**Только ветка:** «Возьми задачу» → Cursor: `Нужна ветка` → ветка → стоп → «реализуй» → код.

**Хотфикс:** «Возьми задачу» → Cursor: `Хотфикс` → сразу правка в Agent mode.

**Баг-репорт:** карточка типа `bagi` в «В работе» → тот же workflow; при закрытии — **anytype-task-finish** обновляет ту же карточку.
