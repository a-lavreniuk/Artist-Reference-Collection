# KB Chrome Contract (Navbar 2.0)

Источник правды для пользовательской документации (Notion Knowledge → Tilda).  
При запросе «обновить knowledge base» по UI chrome — сверять **все** статьи Knowledge (44 leaf + 7 hub + главная), не точечный grep.

## Как описывать пользователю (актуально)

### Структура окна

1. **Top Bar** (24 px): назад/вперёд, **название раздела**, window controls.
2. **Навбар** — **одна** полоса из трёх островков (не «верхняя + нижняя строка»):
   - слева: вкладки **Библиотека / Коллекции / Мудборд / Доска**;
   - центр: поиск (сжатый → расширенный);
   - справа (на `/gallery`, `/collections/*`, `/moodboard`): **сортировка**, **фильтры**, **область библиотеки**, **Добавить**, **⋯**.

### Фильтры

- Кнопка-**иконка**, не текст «Фильтры» и **не** панель под шапкой.
- Клик → меню «Список фильтров» → клик по категории → **второе меню** справа с параметрами.
- Пункты: «Настроить список», «Пресеты», «Очистить фильтры».
- Активные фильтры: точка на иконке.

### Сортировка

- Отдельная кнопка-иконка **рядом** с фильтрами.

### Размер сетки

- **Меню «⋯» → «Размер сетки»** (Большая / Средняя / Маленькая). Не в шапке отдельной кнопкой.

### Тип медиа (изображения / видео)

- Фильтр **«Тип файла»** в меню фильтров. Нет кнопок «Всё / Изображения / Вideo» в шапке.

### Импорт

- Кнопка **«Добавить»** (brand), не «+».

### Доска

- Вкладка **«Доска»** в навбаре, не пункт меню «⋯» и не переключатель во «второй строке» мудборда.

## Запрещённые формулировки в KB

| Устарело | Заменить на |
|----------|-------------|
| две строки шапки / нижняя строка / вторая строка | Top Bar + одна полоса навбара |
| панель «Фильтры» развернётся под шапкой | меню «Фильтры» (иконка), выпадающее |
| кнопка «+» | «Добавить» |
| Всё / Изображения / Видео в шапке | «Тип файла» в фильтрах |
| переключатель размера сетки в шапке | «⋯» → «Размер сетки» |
| Доска в меню «⋯» | вкладка «Доска» |

## Процедура обновления KB

1. Прочитать этот контракт + перечисленные файлы кода.
2. Получить полный список страниц: главная Knowledge + 7 разделов + все `<page>` внутри.
3. Notion search по каждой **запрещённой формулировке** (page_url = Knowledge root).
4. Обновить все hit-страницы.
5. Повторный search — **0** релевантных устаревших hit по navbar/filters.
6. Переопубликовать Tilda вручную (Notion ≠ Tilda автоматически).
