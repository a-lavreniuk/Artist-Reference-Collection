# ARC2 — плоская навигация

Правило: **пункт в сайдбаре → сразу контент**. Без Collection внутри Collection, без Set'ов внутри KB.

## Что pin в сайдбаре (5 пунктов)

| # | Название | Тип | Что открывается |
|---|----------|-----|-----------------|
| 1 | Artist Reference Collection | Page | Home, ссылки на разделы |
| 2 | Задачи | Set (Query) | **Канбан** задач |
| 3 | Баги | Set (Query) | **Канбан** багов |
| 4 | Knowledge Base | Page | Одна страница со ссылками на статьи |
| 5 | Дизайн-система | Page | Одна страница со ссылками на разделы ДС |

**Не pin:** Служебное, шаблоны, старые Collection/Set, отдельные статьи.

## Шаг 1 — Pin

Открой объект → **булавка** справа сверху (или ПКМ → Pin to channel).

ID — в [anytype-config-arc2.md](./anytype-config-arc2.md), раздел «Канал».

Скрыть секцию **Objects** / **Types** в сайдбаре: ПКМ → **Скрыть раздел**. Иначе AnyType показывает каталог всех Pages/Collections/Queries — это не меню проекта.

## Шаг 2 — Канбан «Задачи»

Открой **Задачи** → Edit query → тип **Задача**.  
Вид **Kanban** → группировка **Статус задачи**.

## Шаг 3 — Канбан «Баги»

Открой **Баги** → тип **Баг**.  
Kanban → **Статус бага**.

## Шаг 4 — Страницы KB и ДС

**Knowledge Base** и **Дизайн-система** — обычные **Page**.  
Новую статью: создать «Статья KB» → добавить ссылку на страницу KB вручную (или позже — по запросу автоматизируем).

## Проверка

- Сайдбар: 5 пунктов.
- Задачи / Баги: канбан, не вложенность.
- KB / ДС: одна страница, внутри ссылки — не отдельные Query в сайдбаре.
