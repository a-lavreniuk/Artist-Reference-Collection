## О чём эта статья
Здесь — технические детали установки и обновления ARC. Актуальная публичная сборка на момент обновления базы — **0.1.4** (Windows). Если вы тестируете бета-сборку, начните с неё: так проще понять, откуда брать установщик и чего ждать от автообновления.
## Где взять установщик
Актуальные сборки публикуются на [GitHub Releases](https://github.com/a-lavreniuk/Artist-Reference-Collection/releases). Ищите файл **ARC Setup 0.1.4.exe** (или новее) и скачивайте только с официальной страницы релиза — сторонние зеркала мы не поддерживаем.
## Предупреждение SmartScreen
Windows может заблокировать запуск: у установщика пока нет коммерческой подписи. Если видите предупреждение, нажмите **«Подробнее»**, затем **«Выполнить в любом случае»**. Пожалуйста, отметьте в отчёте, появлялся ли у вас этот экран — это помогает понять, насколько часто тестеры сталкиваются с блокировкой.
## Программа и библиотека — разные места
Папка установки в Program Files и **папка библиотеки**, которую вы выбираете при первом запуске, — это не одно и то же. Переустановка ARC не должна удалять библиотеку, если вы сами явно не удалили её файлы.
## Dev-сборка и установленный ARC
Если вы запускаете проект через `npm run dev`, служебные данные попадают в `%APPDATA%\ARC-dev`, а библиотека по умолчанию называется **«Библиотека ARC (Dev)»**. Установленный релиз использует `%APPDATA%\ARC` и **«Библиотека ARC»**. Профили не пересекаются — можно держать обе копии параллельно, не боясь смешать настройки.
## macOS — тестовые сборки
DMG для macOS пока **не публикуется** автоматически в Releases. Для проверки на Mac используйте артефакт из GitHub Actions (workflow **Build macOS**). Установка: открыть DMG, перетащить ARC в Applications, при первом запуске подтвердить доступ в настройках безопасности macOS. Поведение автообновления на macOS может отличаться от Windows — укажите платформу в баг-репорте.
## Автообновление
При каждом запуске ARC проверяет GitHub Releases. Если есть новая версия и вы соглашаетесь, приложение скачивает её и устанавливает тихо, с перезапуском. После обновления появляется модалка **«Что нового?»**. Файл **latest.yml** в релизе нужен механизму обновления Electron.
При тестировании имеет смысл проверить обновление с предыдущей версии (например **0.1.3 → 0.1.4**), отказ от установки, обновление из иконки в трее и поведение при нестабильной сети.
## Где посмотреть список изменений
В приложении: **«Настройки → Обновления»** и кнопка **«Что нового?»**. В репозитории: [release-notes.json](https://github.com/a-lavreniuk/Artist-Reference-Collection/blob/main/release-notes.json).
