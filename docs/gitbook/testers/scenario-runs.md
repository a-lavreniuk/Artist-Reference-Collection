## Назначение
Сквозные сценарии — как реальный пользователь проходит приложение от начала до конца. Дополняют **Матрицу функционала**, не заменяют её.
 ---
## Сценарий 1: Первый запуск (~20 мин)
**Цель:** новый пользователь, чистая установка.
- [ ] Установка (Windows: SmartScreen; macOS: Gatekeeper)
- [ ] Онбординг: Welcome → Features → создание библиотеки
- [ ] Тур интерфейса завершается без ошибок
- [ ] Импорт 5 файлов (drag-drop)
- [ ] Карточка открывается в оверлее
- [ ] Добавить 1 метку, 1 коллекцию, 1 карточку в мудборд
- [ ] Настройки → AI Поиск: включить, скачать лёгкую модель
- [ ] Дождаться индексации (или зафиксировать статус)
- [ ] Поиск «AI Семантика» — первая выдача
- [ ] Закрыть → открыть из трея — данные на месте
 ---
## Сценарий 2: Ежедневная работа (~30 мин)
**Цель:** типичный день с уже наполненной библиотекой (20+ карточек).
- [ ] Найти карточку: метки → фильтр → сортировка
- [ ] AI-запрос и «По цвету»
- [ ] «Похожие» с карточки
- [ ] Раздел меток: новая категория, перенос метки
- [ ] Коллекция: создать, добавить карточки, reorder
- [ ] Мудборд: добавить/убрать
- [ ] ПКМ на карточке — все пункты
- [ ] Скриншот Ctrl+Alt+E
- [ ] Сохранить с сайта через расширение
- [ ] Статистика и История — данные актуальны
- [ ] Сменить тему, размер сетки
 ---
## Сценарий 3: Перед релизом (~2 ч)
**Цель:** полная уверенность в сборке.
1. Пройти **Сценарий 2** целиком.
2. Отметить **все** пункты **Матрицы функционала** (7 областей).
3. Платформа: Windows smoke + (если есть) macOS smoke.
4. Зафиксировать версию, ОС, RAM, размер библиотеки в отчёте.
 ---

## Сценарий 4: После обновления (~15 мин)
- [ ] Версия до / после
- [ ] «Проверить обновления» → установка → «Что нового?»
- [ ] Библиотека цела, быстрый smoke (импорт, оверлей, поиск)
- [ ] Пункты из changelog текущей версии (см. release-notes)
 ---
## Сценарий 5: Потеря данных и восстановление (~20 мин, опционально)
- [ ] Резервная копия → файл на диске
- [ ] Восстановление на тестовой копии
- [ ] Проверка целостности
- [ ] Перенос папки библиотеки
