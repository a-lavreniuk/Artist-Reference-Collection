Когда **«Библиотека»** разрастается до сотен и тысяч карточек, одной сетки уже мало. Нужны инструменты, которые помогают находить нужное за секунды и собирать подборки под конкретную задачу — без дублирования файлов на диске и без бесконечных папок в проводнике.
ARC предлагает несколько уровней организации. **Метки** и **категории** — гибкая система ярлыков: на одной карточке может быть много меток из разных групп. **«Коллекции»** — именованные подборки под проекты и темы, одна карточка может входить в несколько коллекций сразу. **«Мудборд»** — рабочий набор «на сейчас», когда нужны референсы для текущей сессии, а создавать отдельную коллекцию лень или рано. Дополняют картину **«Статистика»** и **«История»** — сводка по базе и журнал недавних действий.
В этом разделе собраны статьи по каждому инструменту. Начните с обзора, если хотите понять общую логику; если уже знаете, зачем вам метки или коллекции — открывайте нужную тему напрямую.
