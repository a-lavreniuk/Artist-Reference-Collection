## Зачем искать по ID
У каждой карточки в ARC есть **уникальный идентификатор** — длинная строка символов, похожая на UUID. Обычно вы не запоминаете её наизусть, но она незаменима, когда нужно открыть **одну конкретную** карточку: вы скопировали ID из свойств карточки, из переписки с коллегой или из отчёта тестировщика и хотите сразу попасть в нужный референс, минуя пролистывание сетки.
Поиск по ID — не отдельный режим с иконкой, а возможность внутри режима **«Метки»**: та же строка в центре навбара принимает и метки, и идентификатор.
## Как это работает
Убедитесь, что активен режим **«Метки»**. Вставьте или введите полный ID в строку поиска и нажмите **Enter**. ARC найдёт ровно одну карточку с таким идентификатором — если она есть в текущем контексте — и покажет её в сетке. Обычно сетка сужается до одной карточки; из неё можно открыть детальный просмотр.
ID чувствителен к точности: лишний пробел или обрезанная строка не дадут результата. Если карточка не найдена, проверьте, что скопировали идентификатор целиком и что карточка не в другом контексте — например, вы ищете внутри коллекции, куда эта карточка не входит.
## Где взять ID карточки
Идентификатор виден в **свойствах карточки** при детальном просмотре — его можно скопировать оттуда. Тестировщики и разработчики иногда передают ID в баг-репортах, чтобы однозначно указать проблемный референс.
## Контекст и сочетания
На вкладках **«Библиотека»**, **«Коллекции»** и **«Мудборд»** поиск по ID учитывает текущую сетку. В открытой коллекции карточка найдётся только если входит в эту коллекцию. После нахождения можно снять ID из строки и вернуться к обычному поиску по меткам.
Активные **фильтры** справа в навбаре тоже влияют на выдачу: если фильтр исключает найденную карточку, сетка может оказаться пустой даже при верном ID — тогда временно сбросьте фильтры.
