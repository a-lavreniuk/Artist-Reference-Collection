## О чём эта категория
Фильтр **«Описание»** отбирает карточки по тому, **заполнено ли текстовое описание** у референса, и при необходимости — по **ключевым словам** внутри этого текста. Описание — это заметка, которую вы или коллега добавили к карточке: источник, контекст, комментарий для команды.
Категория помогает навести порядок в базе («покажи всё без описания») или найти карточки, где упоминалось конкретное слово — имя художника, проект, техника.
## Режимы «есть» и «нет»
Во втором меню можно выбрать **«Есть описание»** или **«Нет описания»**. Первый вариант оставляет только карточки с непустым текстом; второй — те, где описание ещё не заполнено. Счётчики показывают, сколько таких карточек в текущем контексте.
Это удобный способ для регулярной «уборки»: отфильтровать референсы без описания и постепенно дописать заметки. Или наоборот — быстро найти уже аннотированные материалы.
## Поиск по ключевым словам
При выборе **«Есть описание»** может появиться поле для **ключевых слов**. Введите одно или несколько слов через пробел — ARC оставит карточки, где **все** эти слова встречаются в описании (логика «И»). Подсказка в поле напоминает: «ключевые слова — через пробел, все обязательны».
Фильтр смотрит именно в текст описания карточки, а не в метки и не в имя файла. Если слово есть только в ярлыке, этот фильтр его не найдёт — для смысла лучше поиск по **меткам** в центре навбара.
## Сочетания и контекст
На вкладках **«Библиотека»**, **«Коллекции»** и **«Мудборд»** фильтр складывается с остальными категориями. Часто его комбинируют с **«Ссылка»** — найти карточки, где и описание, и URL заполнены. Или с **«Дата добавления»**, чтобы разобрать свежий импорт без аннотаций.
