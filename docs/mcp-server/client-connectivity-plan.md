# План: подключение MCP-клиентов к ARC

Карточка AnyType: **MCP** (`bafyreidrcf7w2wyx3vl5rhxucypzv5o3wpqhgqcr2z55zkwhueso6ortjq`)  
Ветка: `bug/mcp-client-connectivity`  
Статус документа: **в реализации**

### Зафиксированные решения

| # | Решение |
|---|---------|
| 1 | **Dual transport:** HTTP + native stdio (`ARC --mcp`) |
| 2 | Облачные чаты без localhost — **не делаем** |
| 3 | Основной UX: **копировать пакет** (HTTP + stdio) → вставить агенту |
| 4 | В UI и буфере — **без названий агентов/IDE** и без имён чужих конфиг-файлов |
| 5 | Свой CLI, не `npx` / `mcp-remote` |
| 6 | В буфере — structured текст с абсолютным путём; без готового промпта-установщика |
| 7 | В UI после включения: короткая инструкция RU + только «Копировать», **без** показа JSON |

### UX пользователя

1. Включить MCP в настройках.
2. Нажать «Копировать».
3. В MCP-клиенте попросить установить сервер и вставить буфер.
4. Клиент выбирает HTTP или stdio из пакета.
5. Если агент не пишет файлы — ручная вставка JSON из того же текста.

### Архитектура

- HTTP: `http://127.0.0.1:47897/mcp` (как сейчас).
- Stdio: `ARC.exe --mcp` → `StdioServerTransport` + bridge на localhost HTTP.
- Пакет буфера и запись конфига: `src/main/mcp/mcpSetupClipboard.ts` + IPC.
