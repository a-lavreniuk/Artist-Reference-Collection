# Разные режимы палитры в поиске (HEX / RGB / CMYK / HSL / HSB / Pantone)

Документ-проработка для задачи «Разные режимы палитры в поиске».

## 1. Текущее состояние (после реализации)

| Зона | Поведение |
|------|-----------|
| Бар поиска (режим «Цвет») | `ColorFormatInput` (embedded): переключатель формата + поля ввода + свотч. Форматы: базовые + **Pantone** |
| Search Menu | Пресеты, HSV-палитра (`paletteOnly`), точность. При формате **Pantone** — блок «Ближайшие совпадения» под палитрой |
| Колор-пикер в модалках | `ColorFormatInput` (базовые форматы, **без** Pantone) + HSV-палитра |
| Поиск | Все форматы → HEX → ΔE в Lab (`colorSearch.ts`) |

## 2. UI: переключатель формата

### Форматы

| Формат | Поля ввода | Где доступен |
|--------|------------|--------------|
| HEX | 6 символов | везде |
| RGB | R, G, B (0–255) | везде |
| CMYK | C, M, Y, K (0–100) | везде |
| HSL | H (0–360), S, L (0–100) | везде |
| HSB | H (0–360), S, B (0–100) — совпадает с HSV колор-пикера | везде |
| Pantone | код (`185-C`, `Warm Red C`) | только бар поиска по цвету |

### Механика

Якорь — клик по **prepend-блоку** (`ColorFormatInput`). Dropdown на базе `ContextMenu`. Между каналами в многосоставных режимах — вертикальный сепаратор (Figma [1901:11830](https://www.figma.com/design/JD3pZdlV4Sz62creRMQsJV/ARC-2?node-id=1901-11830)).

Переключение формата **не закрывает** Search Menu, если оно было открыто (клики по `context-menu` исключены из outside-close).

Предпочтение формата: `sessionStorage` (`arc-color-format`).

## 3. Конвертация

| Пара | Точность |
|------|----------|
| RGB ↔ HEX | Точная |
| HEX ↔ CMYK | Математическая, без ICC — **не для печати** |
| HEX ↔ HSL | Стандартная |
| HEX ↔ HSB | = HSV колор-пикера |

Утилиты: `renderer/src/utils/colorFormats.ts`.

## 4. Pantone (Figma [1902-19518](https://www.figma.com/design/JD3pZdlV4Sz62creRMQsJV/ARC-2?node-id=1902-19518))

- **Данные:** `renderer/src/data/pantoneCoated.json` — открытая **приблизительная** таблица Solid Coated (≈400 записей). Не для печати: без ICC и лицензионных данных Pantone.
- **Утилиты:** `renderer/src/utils/pantoneColors.ts` — парсинг кода (пробел/дефис/`pantone`/`pms`), `pantoneToHex`, `findNearestPantones` (ΔE76 в Lab), `formatPantoneCode` (`185 C` → `185-C`).
- **Ввод в баре:** печать кода → резолв в цвет → обычный поиск по HEX. Отдельного выпадающего списка в баре нет. Если код не найден — поле возвращается к ближайшему коду текущего цвета.
- **Ближайшие совпадения:** только при формате Pantone. Под палитрой в Search Menu — до **6** чипов Pantone, ближайших к текущему цвету (`PantoneNearestMatches`). Клик по чипу задаёт цвет (бар показывает его код) и сразу запускает поиск.
- **Проброс формата:** `colorFormat` / `setColorFormat` в `NavbarSearchProvider`; бар — контролируемый, чтобы Search Menu знал формат. Сохраняется в `sessionStorage` (`arc-color-search-format`).

## 5. Логика поиска

**Вариант A:** формат (в т.ч. Pantone) — способ ввода. URL `color=` — HEX. Backend не меняется.

## 6. Данные карточек

`palette_json` остаётся `{ hex, pct }`. RGB/CMYK/HSL/HSB — деривация на лету при отображении. Pantone карточкам не хранится — только подбор в баре поиска.
