# Оценка WD14 / HF-tagger vs JoyCaption+match (ARC)

**Дата:** 2026-07-19  
**Ветка:** `auto-tagging`  
**Фаза 4A:** desk research + методика ручной оценки на библиотеке ARC — **без новой модели в продукте**.

Связано: [auto-tagging-research.md](./auto-tagging-research.md), текущий engine в `src/main/ai/suggestTags.ts`.

---

## 1. Вопрос

Нужен ли ARC отдельный HF-tagger (WD14 / WD-v3 и аналоги), или достаточно JoyCaption + match к каталогу пользователя?

---

## 2. Кандидаты

| Система | Тип | Выход | Язык словаря | Runtime |
|---------|-----|-------|--------------|---------|
| **JoyCaption** (уже в ARC) | VLM | Caption или список меток по prompt | Свободный (у нас — русский) | llama-server (уже есть) |
| **WD14 / WD-tagger v3** (SmilingWolf) | Multi-label classifier | Danbooru-style tags + confidence | Англ. booru (`1girl`, `solo`, …) | ONNX / timm, отдельный пайплайн |
| **JoyTagger** (тот же автор, что JoyCaption) | Tagger | Более «общие» теги, не только anime | В основном EN | Отдельная модель |
| **RAM / RAM++** | Open-set | Много общих категорий | EN | Отдельный runtime |

WD-v3 (ConvNeXt / ViT-Large) — актуальная линейка SmilingWolf; метрики F1 на Danbooru-валидации, не на пользовательском каталоге ARC.

---

## 3. Соответствие продукту ARC

| Критерий | JoyCaption + catalog match | WD14 / WD-v3 |
|----------|----------------------------|--------------|
| Знает **метки пользователя** | Да (exact + CLIP по именам/описаниям) | Нет — чужой словарь |
| Русский каталог | Prompt на русском; exact match сильный | Нужен EN→RU / словарь-маппинг |
| Дубли «портрет / женский портрет» | Контроль prompt + dedupe + пороги | Свои booru-иерархии, другой шум |
| Доп. модель / диск / RAM | 0 (heavy уже для поиска) | +ONNX (~сотни МБ) + код + UI установки |
| Anime / illustration | Средне (VLM) | Сильная сторона WD |
| Photo / concept art / смешанная библиотека | Лучше за счёт открытого словаря | Слабее вне anime-домена |
| MCP / один engine | Уже общий | Второй путь → два поведения |

**Главный разрыв:** ARC оптимизирует **reuse каталога пользователя**, а не полноту Danbooru-тегов. WD14 блестяще решает другую задачу.

---

## 4. Когда WD имел бы смысл

- Библиотека почти целиком anime/illustration **и** пользователи хотят сырой booru-словарь как источник create.
- Готовы держать второй runtime и маппинг EN→RU / synonym table.
- JoyCaption+match стабильно даёт <X% useful matches на смоке (порог продукта).

Пока этих сигналов нет — **не подключать WD в продукт**.

---

## 5. Методика ручной оценки (10–20 карточек)

Без кода WD в репозитории. Цель — зафиксировать, «хватает ли текущего engine».

### Выбор выборки

- 10–15 **изображений** из реальной библиотеки: портрет, среда, стилизация, текст на кадре, несколько объектов.
- 3–5 **видео** (после фазы 4C): короткий клип, смена сцены, почти статичный кадр.
- Разный размер каталога меток: карточка с 0 меток / с 5+ ручными.

### Протокол

1. Включить автотегирование, объём 50%, create выкл. и вкл. (два прогона).
2. Для каждой карточки: «Предложить метки» → записать:
   - число `candidates` / `matched` / `proposedNew` (или созданных);
   - 0–2 полезных метки глазами (да/нет);
   - ложные срабатывания (имена).
3. Опционально вне ARC: прогнать те же файлы через [HF Space WD-tagger](https://huggingface.co/spaces/SmilingWolf/wd-tagger) и сравнить **перекрытие с вашим каталогом** (не «красоту» EN-тегов).

### Шаблон строки

| cardId | тип | matched useful | false positive | create полезен? | заметка |
|--------|-----|----------------|----------------|-----------------|---------|
| … | image/video | 0–N | … | да/нет | … |

### Вердикт (заполнить после смока)

- [ ] JoyCaption+match достаточен для v1–v2  
- [ ] Нужен только лучший prompt / пороги  
- [ ] Имеет смысл пилот WD как **опциональный** backend (отдельная задача)

**Предварительный вердикт (desk, до смока):** WD **не** добавлять в ARC сейчас; усиливать match и video-frames. Пересмотреть после заполнения таблицы на ≥10 карточках.
