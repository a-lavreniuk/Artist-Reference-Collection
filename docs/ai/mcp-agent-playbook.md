# MCP playbook: агент и автотегирование ARC

Краткий сценарий для внешнего MCP-клиента. Подключение сервера: [docs/mcp-server/README.md](../mcp-server/README.md).

Связанное исследование: [auto-tagging-research.md](./auto-tagging-research.md).

## Предусловия

1. ARC запущен, библиотека открыта.
2. **Настройки → MCP сервер** — сервер включён, инструмент `arc_suggest_card_tags` разрешён.
3. **Настройки → AI Поиск** — AI включён, установлена тяжёлая модель (JoyCaption).
4. **Настройки → Автотегирование** — автотегирование включено.

Без пунктов 3–4 tool вернёт понятную ошибку, а не «пустой» ответ.

## Один engine с приложением

| В UI | В MCP |
|------|--------|
| Кнопка «Предложить метки» в деталке | `arc_suggest_card_tags` |
| Тоггл «После импорта…» | тот же `suggestTagsForCard` после heavy-индекса |

Агент **не** должен угадывать метки «на глаз» по thumb URL, если доступен `arc_suggest_card_tags`.

## Инструмент `arc_suggest_card_tags`

**Вход:** `{ cardId }` (изображение или видео).

**Выход (read-only):**

```json
{
  "cardId": "…",
  "matched": [{ "tagId": "…", "name": "портрет", "score": 1, "via": "exact" }],
  "proposedNew": ["киберпанк"],
  "candidates": ["портрет", "киберпанк", "…"],
  "tagIds": ["…"]
}
```

- `matched` / `tagIds` — уже есть в каталоге пользователя (exact или CLIP).
- `proposedNew` — кандидаты без match. MCP **никогда не создаёт** метки сам (`allowCreate: false`).
- Карточку tool **не меняет**.

## Prompt `suggest_tags`

В MCP зарегистрирован шаблон `suggest_tags`: предложить → показать → спросить → только потом писать.

## Рекомендуемый workflow

1. Найти карточку (`arc_search_cards` / `arc_list_cards` / `arc_ai_search`).
2. Вызвать `arc_suggest_card_tags` с `cardId`.
3. Показать пользователю `matched` и `proposedNew`.
4. После подтверждения:
   - прочитать текущие метки (`arc_get_card`);
   - **merge** `tagIds` (не replace всего списка);
   - вызвать `arc_set_card_tags` с полным итоговым списком.
5. Для `proposedNew`: спросить, создавать ли метки (`arc_create_tag` в категорию пользователя или «Автоматически созданные метки»), затем снова merge.

## Пример запроса агенту

```
Для карточки <cardId> предложи метки через arc_suggest_card_tags.
Покажи список matched и proposedNew. Ничего не записывай, пока я не подтвержу.
```

## Типовые ошибки

| Сообщение | Что сделать |
|-----------|-------------|
| AI Поиск выключен / автотегирование выключено | Включить в настройках ARC |
| Нужна тяжёлая модель | Скачать JoyCaption в AI Поиск |
| Только для изображений | Устаревшее — видео поддерживается через кадры |
| Не удалось извлечь кадр | Проверить ffmpeg / файл видео |
| Модель занята | Дождаться индексации, повторить |

## Связанные tools

- `arc_set_card_tags` — запись (полная замена списка на карточке; merge делайте на стороне агента).
- `arc_list_tags` / `arc_create_tag` — каталог.
- `organize_imports` — массовая разметка импорта; для картинок предпочитать suggest engine.
